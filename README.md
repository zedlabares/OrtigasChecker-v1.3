# POS File Checker

A zero-dependency static web application for validating POS `.001` remittance files.

**No npm. No Node.js. No build step. Open `index.html` and it works.**

## What it does

Uploads a `.001` POS terminal remittance file, parses all 65 fixed-width line items, re-derives the expected values using back-office VAT accounting rules, and compares the two — producing **Pass** or **Failed** on 11 critical checkpoints. Faithfully replicates the VBA macros in `POSFileChecker_v1.xlsm`.

**All processing runs locally in the browser. The file never leaves the device.**

## Run locally

No server required for development. Simply open `index.html` directly in your browser:

```bash
# macOS
open index.html

# Linux
xdg-open index.html

# Windows
start index.html
```

Or serve with any static server if you want the service worker to activate (service workers require HTTP, not `file://`):

```bash
# Python (built-in, no install needed)
python3 -m http.server 8080
# → http://localhost:8080

# Node (if you happen to have it)
npx serve .
```

## Business logic

All calculations are in `script.js`, Section 4 (Calculator). Ported directly from the VBA macros.

### The 11 validated fields

| Line | Field | Formula |
|------|-------|---------|
| 5  | New Accumulated Sales (VAT)       | `OldAcc/100 + (Gross−Ded)/1.12/100` |
| 6  | Total Gross Amount (VAT)          | `Gross/100` *(structural identity)* |
| 7  | Total Deductions (VAT)            | `Σ lines 7–22 / 100` |
| 24 | Total Non-Approved Discounts      | `Σ lines 25–29 / 100` |
| 30 | Total VAT/Tax Amount              | `(Gross−Ded) × 12/112 / 100` |
| 31 | Total Net Sales (VAT)             | `(Gross−Ded) / 1.12 / 100` |
| 36 | Amount (Non-VAT section)          | Same as line 31 |
| 38 | New Accumulated Sales (Non-VAT)   | `OldAccNV/100 + (GrossNV−DedNV)/100` |
| 40 | Total Deductions (Non-VAT)        | `Σ lines 41–56 / 100` |
| 64 | Total Net Sales (Non-VAT)         | `(GrossNV−DedNV) / 100` |
| 65 | Grand Total Net Sales             | `NetVAT + NetNonVAT` |

Pass/Fail comparison rounds both values to 2 decimal places before comparing.

### Known anomaly — Line 57

Line item 57 (Non-VAT Total Non-Approved Store Discounts) was written to Column D without the standard ÷100 division in the original VBA workbook. This is replicated faithfully. The field is not validated. The UI shows a ⚠️ warning tooltip.

### File encoding

UTF-8 → Windows-1252 (ANSI) → ISO-8859-1 fallback, handled by `TextDecoder` in `script.js`.

## Export

- **PDF** — generated by [jsPDF](https://github.com/parallax/jsPDF) + AutoTable, loaded from cdnjs at export time
- **Excel (.xlsx)** — generated by [SheetJS](https://sheetjs.com/), loaded from cdnjs at export time

Both libraries are loaded on demand (only when the user clicks Export), so they don't affect the initial page load. If offline, export buttons show an alert and the core validation still works.

## PWA

After the first visit the service worker (`sw.js`) caches the app shell. The app loads and validates files fully offline. Install via Chrome/Edge → address bar → install icon, or Safari → Share → Add to Home Screen.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Blank page on GitHub Pages | Settings → Pages → Source must be **GitHub Actions** |
| 404 on hard refresh | `404.html` must be present in the repo root — do not delete it |
| Export buttons say "library loading" | You are offline; reconnect once to cache the CDN libraries |
| File rejected | Only `.001` extensions are accepted; check the file name |
| Wrong values | Verify the file is the correct POS `.001` format (65 lines, fixed-width) |
